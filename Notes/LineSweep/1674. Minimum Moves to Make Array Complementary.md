## What we are solving

`nums` has even length. Index `i` is paired with `n - 1 - i`. After moves, **every pair must have the same sum** `T` (each value stays in `1 … limit`).

The real question:

> Pick one **target sum `T`** that **minimizes total moves** across all `n/2` pairs.

Only mirror pairs matter. For each pair, ask: *how many moves to make this pair sum to `T`?* Add those costs and minimize over `T`.

**Example** (`limit = 6`, see diagram §①): `[1,2,4,3,5,2,6,1]` → pairs `(1,1), (2,6), (4,2), (3,5)`.

---

## Thought process (follows the diagram)

### §① Mirror pairs

Four independent pairs; they must all hit the **same** `T`. Search space = target sums `T ∈ [2, 2·limit]`, not individual indices.

### §② Sliding `T` — right picture, wrong shortcut

- One row per pair; slide a vertical line along possible sums `T`.
- At each `T`, **add** how many moves each pair needs.
- **Dead end:** put `T` where **most pairs already sum to** (mode / heap). Current sums `2,8,6,8` suggest `T=8`, but **frequency ≠ minimum cost**.
- **Keep:** sliding `T` and minimizing **total moves**, not counting bar ends.

### §③ One pair, three targets

Panel ③: pair `(2,6)` only.

| Target `T` | Idea | Moves |
|------------|------|-------|
| 8 | Already `2+6` | **0** |
| 6 | Change one side | **1** |
| 2 | Change both sides | **2** |

Cost depends on `T`, not on “where the pair is today” in isolation.

### §④ Whole array — naive answer

1. **One `T` for all** — not four different targets.
2. **Add** pair costs — e.g. `T=6`: `1+1+0+1 = 3` moves.
3. **Compare** — `T=8` also totals **3**; `T=4` totals **4**. **Answer = 3.**

Trying every `T` and summing all pairs is **correct** but **O(limit × n)** when `limit` is large.

### §⑤ Why line sweep?

- Each pair’s `cost(T)` is **flat between a few breakpoints** (0 / 1 / 2 move bands).
- **Many pairs × few stamps** on the `T` axis → **difference array + prefix walk** for `min total(T)`.
- Same pattern family as 1854 and meeting rooms. Implementation: [[Line Sweep general theory]].

## Recognition checklist

1. Find the **shared parameter** (here `T`).
2. Write **cost per item as intervals** on that line — not a histogram of current values.
3. Need **min or max of a sum** over many interval updates → **line sweep**.

---

## Wrong vs right

| Idea | Verdict |
|------|---------|
| Mirror pairs; one global `T` | ✓ |
| Mode / heap on current sums | ✗ — ignores move cost per `T` |
| Per-pair 0 / 1 / 2 cost vs `T` | ✓ |
| `diff[]` + prefix sweep | ✓ |
