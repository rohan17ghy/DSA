Sorted sets ^617ph54n

Typical implementation: balanced binary search tree (e.g. red–black, AVL) — O(log N) insert / delete / lookup
when N is the number of keys. 

Traversal order matches sorted order. ^1T17idZr

1) Ordering invariant (why it is “sorted”) ^9vzU6yjI

Keys are kept in sorted order at all times.
• left subtree  <  node key  <  right subtree  (BST rule)
• An in-order walk (left → node → right) visits keys from smallest to largest. ^7euLhgCB

2) Backbone: balanced BST stores the keys ^kj982fUD

Example tree (heights exaggerated for teaching — real libs keep it balanced): ^4FG4bCh6

40 ^yWV5zNdL

25 ^3tZgEp5h

55 ^R8LwabaM

10 ^Z04ze4MK

30 ^B1EEF4YL

50 ^sNxPZFUD

70 ^E1nTr5ww

In-order: 10 → 25 → 30 → 40 → 50 → 55 → 70  (sorted) ^MOoWBa5O

3) Search (contains / lower_bound style) ^aj9e1zqc

Start at root.
• key < node  → go left
• key > node  → go right
• equal       → found
Descend at most tree height → O(log N) if balanced. ^Aubiv8PL

Example: search 30
40 → too big → left
25 → too small → right
30 → match
Path length ≈ height. ^u1ZpSKtH

(Libraries may use parent pointers or iterative walk — same comparison rule.) ^ZvHn57Et

4) Insertion ^xRZcllee

1) Search as if the key were already there — fall off at an empty child link.
2) Attach new leaf with the key.
3) Rebalance / recolor (red–black rotations, AVL fixes, …) so height stays O(log N). ^5uU7ABmE

Duplicate policy: some structures allow duplicates (with counts); std::set-style types usually reject duplicates. ^oOHMVYlQ

5) Smallest / largest / top‑K by order ^sj3eSu85

• Minimum: repeatedly take left child from root — O(log N) (or store pointer to leftmost).
• Maximum: symmetric, go right.
• Top‑K smallest (total keys = N): walk in-order from the minimum — stop after emitting K keys — typically
O(K + log N) (reach far-left in O(log N), then K in-order steps). ^ASgX6TrP

Example: K = 3 → first three keys in-order: 10, 25, 30. ^6LTTafvw

6) Other real-world structures ^ZvEldpGV

• Skip list (Redis sorted sets): probabilistic levels — expected O(log N) operations; different but similar use cases.
• Sorted array + binary search: O(log N) lookup, O(N) insert — fine for read-mostly data.
• B-trees / B+ trees: great on disk — used in databases and file indexes. ^9rFAGrnD
