This problem is similar to the [[153. Minimum in Rotated Sorted Array]], the only extra thing is to handle the duplicates.

Checkout the below conditions in the code to handle the duplicates. That is the only extra code

```python title:findMin.py hl:22-29
class Solution:

    def findMin(self, nums: List[int]) -> int:

        N = len(nums)
        start, end = 0, N-1
        
        while start < end:

            mid = (start + end) // 2
            
            #Edge case: when start, end and mid are not unique indices.
            #when mid is calculated to be equal to start
            #possible only when end = start + 1.
            #this is done to avoid TLE
            if end == start + 1:
                if nums[end] < nums[start]:
                    start = end
                else:
                    end = start

            #Handle duplicates: when numbers are equal
            elif nums[mid] == nums[start] == nums[end]:
                start = start + 1
                end = end - 1
            elif nums[mid] == nums[start]:
                start = start + 1
            elif nums[mid] == nums[end]:
                end = end - 1

            #2 cases possible
            #First: when mid is towards the left of the smallest number
            elif nums[mid] > nums[start] and nums[mid] > nums[end]:
                start = mid
            #Second: when mid is towards the right of the smallest number
            else:
                end = mid

        return nums[start]
```
